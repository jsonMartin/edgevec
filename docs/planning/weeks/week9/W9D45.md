# EdgeVec FAQ

## Installation

### Q: How do I install EdgeVec?

```bash
npm install @edgevec/core
```

### Q: What browsers are supported?

EdgeVec supports all modern browsers with WebAssembly support:
- Chrome 57+
- Firefox 52+
- Safari 11+
- Edge 16+

### Q: Does it work with Node.js?

Yes! EdgeVec works in Node.js 16+ with full feature parity.

---

## Usage

### Q: What embedding dimensions are supported?

EdgeVec supports any dimension from 1 to 4096. Common dimensions:
- 384 (MiniLM)
- 768 (BERT, all-mpnet-base-v2)
- 1536 (OpenAI text-embedding-ada-002)
- 3072 (OpenAI text-embedding-3-large)

### Q: How do I persist data?

```javascript
// Save to IndexedDB
await client.save('my-index');

// Load later
const client = await EdgeVecClient.load('my-index', { dimensions: 768 });
```

---

## Performance

### Q: Why is my search slow?

Check:
1. Are you using Quantized mode? (3x faster than Float32)
2. Is ef_search set appropriately? (higher = slower but more accurate)
3. Are you running in development mode? (release builds are faster)

### Q: How much memory does it use?

| Mode | Per Vector (768d) |
|:-----|:------------------|
| Float32 | ~3,176 bytes |
| Quantized (SQ8) | ~872 bytes |

For 100k vectors: ~83 MB (Quantized) or ~303 MB (Float32)

---

## Troubleshooting

### Q: Import fails with "WebAssembly.instantiate" error

This usually means WASM isn't supported or is blocked. Check:
1. Browser supports WebAssembly
2. No Content-Security-Policy blocking WASM
3. Running on HTTPS (required for some browsers)

### Q: "Out of memory" error

EdgeVec uses WebAssembly linear memory which can grow but not shrink.
For large datasets (>500k vectors), consider:
1. Using Quantized mode (3.6x less memory)
2. Periodic index reload to defragment
